Security Policy
Last updated: July 2026
Supported Versions
The DubbiOvi project follows a rolling release model.
Security updates are provided for the most recent public release available on GitHub. Users are encouraged to keep their installation up to date.
Version	Supported
Latest release	✅ Yes
Previous releases	⚠ Limited support
Older releases	❌ No
 
Reporting a Security Vulnerability
If you believe you have discovered a security vulnerability in DubbiOvi, please report it responsibly.
Please do not disclose the vulnerability publicly before it has been assessed and, where appropriate, fixed.
Security reports should include, whenever possible:
•	a description of the issue;
•	the affected version;
•	operating system;
•	steps to reproduce the vulnerability;
•	screenshots or logs (if applicable);
•	possible impact.
Reports may be sent to:
Email: rodriguezalfonso@uniovi.es
 
Response Process
Every report will be reviewed as soon as reasonably possible.
Our general process is:
1.	Acknowledge receipt of the report.
2.	Assess the vulnerability.
3.	Reproduce and verify the issue.
4.	Develop and test a fix.
5.	Publish an updated release when necessary.
6.	Credit the reporter, if they wish to be acknowledged.
 
Scope
This policy applies to:
•	the DubbiOvi desktop application;
•	the official GitHub repository;
•	the official project website;
•	associated documentation.
It does not apply to third-party services (such as external AI providers), which are governed by their own security and privacy policies.
 
Responsible Disclosure
We kindly ask security researchers to:
•	act in good faith;
•	avoid accessing data that do not belong to them;
•	avoid disrupting services;
•	provide sufficient information to reproduce the issue;
•	allow reasonable time for remediation before public disclosure.
 
Security Philosophy
DubbiOvi has been designed with the following principles:
•	Local-first architecture
•	Privacy by design
•	Open-source transparency
•	Minimal data collection
•	Secure communication over HTTPS
•	No telemetry or hidden tracking
Security is an ongoing process. Community feedback is always welcome and helps improve the project.
 
Thank You
We sincerely appreciate responsible vulnerability reports and the contributions of the security community in helping keep DubbiOvi safe for everyone.
