Data Management Plan (DMP)
Project: DubbiOvi – Open-Access AI-Assisted Audiovisual Translation Platform
Institution: University of Oviedo (Spain)
Project Lead: Alfonso C. Rodríguez Fernández-Peña
Version: 1.0
Last updated: July 2026
 
1. Purpose
This Data Management Plan (DMP) describes how data are collected, managed, stored, protected, shared, and preserved throughout the lifecycle of the DubbiOvi project.
DubbiOvi has been designed following the principles of Open Science, transparency, privacy by design, and local-first computing.
 
2. Types of Data
The project manages two distinct categories of data.
2.1 Administrative Data
Users who voluntarily register to download DubbiOvi may provide:
•	First name
•	Last name
•	Email address
•	Institution
•	Country
•	Operating system
•	Software version
•	Registration timestamp
These data are used exclusively for project administration and dissemination.
 
2.2 Software Data
DubbiOvi processes audiovisual translation materials locally on the user's computer.
These may include:
•	audiovisual files
•	subtitle files
•	translation memories
•	scripts
•	translated text
•	timing information
•	exported projects
These files remain under the control of the user.
The project does not collect, upload, monitor, or archive these materials.
 
3. Data Collection Principles
DubbiOvi follows the principle of data minimisation.
Only information strictly necessary for project management and communication is collected.
No behavioural analytics, telemetry, tracking technologies, or advertising identifiers are implemented.
 
4. Storage
Administrative data are stored using secure Google Workspace services managed by the project administrator.
Research materials created by users remain on their own devices unless they explicitly choose another storage solution.
 
5. Security
The project applies reasonable technical and organisational safeguards, including:
•	restricted administrative access;
•	password-protected institutional accounts;
•	encrypted communications (HTTPS);
•	regular software updates;
•	secure version control through GitHub.
Users remain responsible for securing the files stored on their own systems.
 
6. External AI Services
Certain optional functions may communicate with third-party AI providers selected by the user.
DubbiOvi acts only as an interface for these requests.
The project does not retain prompts, translated content, or generated outputs exchanged with external AI services.
Users should review the privacy policies of the AI providers they choose to use.
 
7. Data Sharing
Personal information is never sold or disclosed to third parties.
Aggregated and anonymised statistics may be reported in:
•	scientific publications;
•	conference presentations;
•	institutional reports;
•	funding applications;
•	impact and knowledge-transfer assessments.
No individual user will be identifiable in these reports.
 
8. Data Retention
Administrative information is retained only for as long as necessary to support the project and communicate relevant updates.
Users may request deletion of their personal information at any time.
 
9. Open Science
DubbiOvi supports Open Science by providing:
•	open-access software;
•	publicly available documentation;
•	transparent development;
•	version-controlled source code;
•	reproducible releases through GitHub and Zenodo.
Whenever possible, software outputs and documentation are distributed under permissive open-source licences.
 
10. FAIR Principles
The project is committed to the FAIR principles:
•	Findable through GitHub and Zenodo.
•	Accessible as open-access software.
•	Interoperable using widely adopted file formats.
•	Reusable under the MIT License.
 
11. GDPR Compliance
DubbiOvi has been designed to align with the principles established by the General Data Protection Regulation (EU Regulation 2016/679), including:
•	lawfulness;
•	fairness;
•	transparency;
•	purpose limitation;
•	data minimisation;
•	accuracy;
•	storage limitation;
•	integrity and confidentiality.
